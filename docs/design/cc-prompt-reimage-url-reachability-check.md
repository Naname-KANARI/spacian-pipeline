# 報告: reimage.tsが「同じ壊れたURL」を正常判定してしまうケースがある

作成: 2026-07-19
種別: 報告・改善検討依頼

---

## 経緯

`bigger-launch-vehicles-not-always-better-study`のheroImage修復（Deploy #54でのreimage-dispatch実行）を確認したところ、`reimage.ts`が検索の結果、**修復前とまったく同じNASA画像URL**（`https://images-assets.nasa.gov/image/PIA16625/PIA16625~medium.jpg`）を「✓ OK」として採用していた。実際にはこのURLは403のままで、ライブサイトで画像読み込み失敗を確認した。

## 気になる点

`reimage.ts`の判定基準が「NASA検索APIが一致する画像エントリを返したか」までで、**解決後のURLが実際にHTTPで到達可能かの最終確認をしていない**ように見える。同じ検索クエリを投げれば同じ画像が再び見つかるのは自然なので、healthcheck-images.tsが検出した「アクセス不能」という状態を、reimage側では再現・検知できず、同じ壊れたURLを結果として書き戻してしまう。

急ぎの話ではないので、対応するかどうか・するならどう直すか（例: 採用前にHEADリクエストで到達性を確認し、ダメなら次の候補にフォールバックする等）はCCの判断に任せたい。

## お願いしたいこと

このケースについてどう考えるか、対応するとしたらどんな方針が良さそうか、意見を聞かせてほしい。急ぎではない。
